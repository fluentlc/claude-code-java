10 项核心能力

✅

TodoManager

Agent 自追踪任务 — 多步骤任务状态流转（pending → in_progress → completed），超过阈值自动注入提醒。

🧠

ContextCompactor

三层上下文压缩管道：微压缩（每轮替换旧 tool_result）→ 自动全量压缩（消息数阈值）→ 手动 /compact。

⚡

BackgroundRunner

线程池异步执行耗时命令，任务完成后自动注入 system-reminder 通知，不阻塞主对话。

💾

TaskStore

JSON 文件持久化任务 + 有向无环依赖图。重启后自动恢复。支持 task_poll 轮询认领。

🌿

WorktreeManager

Git Worktree 目录级隔离 — 每个特性分支独立工作区，wt_run 在隔离目录执行命令。

🎯

SkillLoader

按需加载技能 — 扫描 skills/ 目录，模型主动调用 load_skill 时注入 prompt，不预占 context。

📨

MessageBus

JSONL 文件消息总线实现多 Agent 协作，lead/worker 通过 msg_send / msg_read 异步通信。

🗂️

SessionStore

REST 会话持久化 — 每个 sessionId 对应 .sessions/{id}.json，进程重启后会话历史自动恢复。

🔒

SecurityUtils

危险命令黑名单（rm -rf /、sudo …）+ 路径遍历防护，bash 工具和 wt_run 均受保护。

🤝

TeammateRunner

Agent Teams 核心 — Lead 通过 spawn_teammate 动态创建 Teammate，每个 Teammate 在独立线程中运行完整 LLM 循环，通过 MessageBus 通信，通过 TaskStore 自主认领任务，60s 无工作自动退出。

快速上手

1. 配置 API Key

编辑 claude-code-4j-start/src/main/resources/claude.properties，填入 API Key：

OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_BASE_URL=https://api.openai.com
OPENAI_MODEL_ID=gpt-4o

或通过环境变量覆盖（优先级最高）：

export OPENAI_API_KEY=sk-xxxxxxxx

2a. CLI 交互模式

进入 REPL 交互式终端：

# 编译
mvn compile

# 启动 CLI（REPL）
mvn exec:java \
  -pl claude-code-4j-start \
  -Dexec.mainClass="ai.claude.code.Application" \
  -Dspring.profiles.active=cli

2b. REST API 模式

启动 HTTP 服务器（默认端口 8080）：

# 启动 REST 服务器
mvn exec:java \
  -pl claude-code-4j-start \
  -Dexec.mainClass="ai.claude.code.Application"

# 调用 API
curl -X POST http://localhost:8080/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message": "你好"}'

Agent Teams 示例

Lead 动态创建 Teammate，Teammate 在独立线程中运行真实 LLM 循环：

# 对 Lead 说（CLI 或 REST 均可）：
请创建两个 Teammate：
- reviewer：负责代码审查
- tester：负责编写单元测试

# Lead 会自动调用：
spawn_teammate(name="reviewer", role="代码审查")
spawn_teammate(name="tester",   role="单元测试")

# 查看 Teammate 状态：
list_teammates()
# → [{"name":"reviewer","status":"working"},...]

# Teammate 自主认领任务：
# Lead 创建 task → Teammate task_poll 认领
# → 完成后 msg_send 回报 lead
